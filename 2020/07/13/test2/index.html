c++前向声明

场景一

C++在调用任何函数之前，必须先看到声明

#include "xx.h" 是一种声明集合，但一次编译所有（包括不需要的），很耗时
这时在顶部用前向声明可以缩短编译时间

场景二

两个定义互相调用，打破循环

//car.h
#include "wheel.h"
Car{
    Wheel wh1;
}
//wheel.h
class Car;	//前向声明
Wheel{
    Car  car1;
}

用递归还是循环

递归是在一个函数的内部调用这个函数自身

递归的优点

通常代码更简洁，容易实现

递归的缺点

  • 函数调用造成时间和空间的消耗

每次函数调用，要在内存栈分配空间保存：参数，返回地址，临时变量

压栈和弹出栈都要消耗时间

  • 可能存在很多重复计算

    如：斐波那契数列的计算
    用自上而下的循环更有效率

  • （基于上两个）调用栈溢出

    如果递归调用的层级太多，就会超出栈的容量
